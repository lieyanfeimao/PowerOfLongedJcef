代码模板生成器帮助文档

本程序可用于根据模板自动生成代码，如有需要，可改进。

场景： 当我们做一些繁琐又重复的文本处理时(比如开发时使用三层架构，每次做新功能可能都要建service,dao等基础层)，我们会觉得枯燥、麻烦。这时候，我们可以提取一个模板，然后，批量处理！

本程序逻辑很简单，读取模板文件->替换指定字符串->生成文件到指定目录。界面仅做数据处理，具体逻辑请参考com.xuanyimao.polj.modeldeal.jsimpl.modelDealImpl

注意：可能会因为某些不小心的操作导致原文件被替换，建议建一个专用目录，将文件生成到该目录后再手动复制过去替换。数据无价，谨慎操作！

操作流程

在项目根目录下，app/modeldeal/model下有一个测试用的模板文件：{entity}Service.java

1.点击新建模板，新建一个模板。每个模板对应一套模板文件及配置，可复制。

2.点击“添加模板文件”，选择模板文件（可多选）

3.点击“编辑”或者双击单元格，设置目标文件路径(文件生成路径)。一般我们会把它生成到项目的某个目录下，我这里把它保存到d:\service\，因为路径可能会变化，配置成关键词：{pj}

4.配置关键词。关键词即模板文件中的 {date}、{entity}这些字符串，可以理解为设置的一些特殊字符串，你可以自定义格式，只要不和模板文件的中不需要替换的内容有冲突就行。
  添加一个关键词{pj}，替换的文本为：d:\service，可以看到，模板文件配置下的目标文件真实路径已发生改变。
  将模板文件中的关键词一一设置好。

5.点击“生成目标文件”，去目标文件夹查看生成是否无误(切记不要做直接替换工程代码这样的危险操作，建议将文件生成到指定目录后再复制过去替换)
